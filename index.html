核心功能

面向小白设计，无需记复杂命令，小白也能轻松上手。

🕰️

标准 NTP 客户端

遵循 RFC5905，用 4 时间戳计算 offset/delay，毫秒级校准精度。可指定任意 NTP 服务器。

🌐

内网 HTTP 时间源

支持 JSON / RFC3339 / Unix 时间戳，并兼容普通 Web 服务器的 Date 响应头（如 nginx）。

🔁

主备链 -chain

按顺序尝试多个时间源，某项失败自动切换：主 NTP + 备 HTTP，或多 NTP 主备轮换。

🚀

开机自动启动

通过 schtasks 注册系统级 onstart 计划任务（SYSTEM 账户），无需登录用户即可运行。

🧩

双协议时间源（server）

本机可同时充当 NTP（UDP 123）与 HTTP（/time）时间源，并后台自校准，供内网其它机器同步。

🗣️

中文配置向导

内置 config.bat，双击后全中文提示选择运行方式与时间源，自动生成并注册开机任务。

A↔B 拓扑：一台既当源又同步上游

软件 A 以 server 模式运行 —— 对内提供 NTP(UDP) + HTTP 时间，后台用 NTP 自校准本机时钟； 软件 B 用标准 NTP（或 HTTP）直接同步 A。完整闭环，无需外网也能构建内网时间基准。

💡 实测拓扑：A 机既对内供时又后台自校准，B 机直接用 NTP 同步 A，偏移 ≈ 0ms：
# A 机：启动 NTP(UDP) + HTTP 双协议时间源 winTimeSync.exe server -server-addr :18090 -server-ntp-port 123 -server-ntp=false # B 机：用 NTP 协议同步 A winTimeSync.exe once -source ntp -ntp-server <A的IP>:123 -check # → 命中 ntp:A，偏移 ≈ 0ms

命令速查

所有写系统时间与 install 注册开机任务都需以管理员身份运行。

常用命令

# NTP，每 10 分钟同步 winTimeSync.exe run -source ntp -interval 600 # 内网 HTTP 源，每 60 秒同步 winTimeSync.exe run -source http -http-url http://127.0.0.1:8080/time -interval 60 # 主用 NTP，备用 HTTP winTimeSync.exe run -chain "ntp:pool.ntp.org:123,http:http://127.0.0.1:8080/time" -interval 60 # 只检查偏差不改时间 winTimeSync.exe once -source ntp -check

双协议时间源 & 开机启动

# A：同时提供 NTP + HTTP（需管理员） winTimeSync.exe server -server-addr :8080 -server-ntp-port 123 -server-ntp=true # nginx 直接提供 /time（无需本软件） # location = /time { # return 200 '{"time":"$time_iso8601"}'; # } # 注册 / 查看 / 移除 开机启动 winTimeSync.exe install -chain "ntp:pool.ntp.org:123,http:http://127.0.0.1:8080/time" -interval 60 winTimeSync.exe status winTimeSync.exe uninstall

参数说明

参数说明默认
-source时间源：ntp | httpntp
-chain主备链，逗号分隔；项格式 ntp:地址 / http:地址空
-ntp-serverNTP 服务器地址pool.ntp.org:123
-http-urlHTTP 时间地址（默认内网）http://127.0.0.1:8080/time
-interval同步间隔（秒），run 模式生效3600
-timeout单次请求超时（秒）5
-check仅检查偏差，不修改系统时间false
-server-ntp-portserver 模式 NTP 监听端口123
